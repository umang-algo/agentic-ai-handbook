Tool Use & Sandboxing

How Claude and ChatGPT Run Code

When you ask ChatGPT to "analyze this CSV," it actually writes a Python script, runs it, and reads the output. This is possible through Function Calling and MicroVMs.

1. Function Calling (Tool Use)

LLMs are fine-tuned to output structured JSON when they realize they need to perform an action. For example:


{
  "name": "python_eval",
  "arguments": {
    "code": "import pandas as pd\ndf = pd.read_csv('data.csv')\nprint(df.head())"
  }
}

The orchestrator script intercepts this JSON, runs the code in a sandbox, and passes the terminal output back to the LLM as the next message.

2. Safe Execution (The Sandbox)

You cannot safely run AI-generated code directly on a host machine (due to security risks like rm -rf /). Platforms like Claude (Artifacts) and ChatGPT (Code Interpreter) solve this using sandboxing:

  • WebAssembly (Wasm) / WebContainers: Running Node.js or Python directly in the user's browser, fully isolated from the host machine. (e.g., Claude Artifacts)
  • MicroVMs (Firecracker): Spinning up a tiny Linux virtual machine in milliseconds on the server, running the code, and destroying the VM immediately after. (e.g., ChatGPT Advanced Data Analysis)
