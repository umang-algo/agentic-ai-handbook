Model Context Protocol (MCP)

The "USB-C" of AI Agents

Currently, integrating an AI agent with a database, a Slack workspace, or an API requires writing custom glue code for every single service. The Model Context Protocol (MCP) aims to standardise this.

How MCP Works

MCP acts as a universal bridge. Instead of hardcoding API logic into your agent, you run an MCP Server. The agent connects to this server and dynamically asks: "What tools and resources do you have available?"

Resources

Data exposed to the model (e.g., "Here is a list of all Jira tickets").

Tools

Executable actions exposed to the model (e.g., "I can update a Jira ticket status").

Prompts

Pre-defined system prompts provided by the server to set the agent's context.

Impact on IDEs

Cursor and Claude Desktop both support MCP. This means you can build a small Python MCP server that connects to your company's internal documentation database, and Cursor will instantly know how to query it to write better code, without you modifying Cursor itself.
